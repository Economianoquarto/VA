Dados e Preparação Geográfica

Este painel interativo apresenta a participação da agropecuária, da indústria e dos serviços no Valor Adicionado municipal em 2021. Para tornar o deploy mais estável, a coleta dos dados e o cruzamento com a malha municipal foram separados em um script externo, chamado 01_preparar_dados.R.

O aplicativo abaixo não baixa dados do SIDRA nem baixa a malha do geobr durante a inicialização. Ele apenas carrega o arquivo leve my_workspace_light.RData, que deve estar no mesmo diretório deste documento.

Como gerar os dados usados pelo aplicativo

Antes do deploy, execute uma vez no console do R, dentro da pasta do projeto:

source("01_preparar_dados.R")

Esse comando cria o arquivo my_workspace_light.RData, contendo apenas os objetos necessários para o painel: br_map e lista_estados.

Inicialização leve do Shiny

library(shiny)
library(dplyr)
library(ggplot2)
library(sf)
library(patchwork)
library(scales)

load("my_workspace.RData")

Arquivos necessários para o deploy

Para publicar este painel em uma plataforma compatível com Shiny, envie ao menos os seguintes arquivos no mesmo diretório do projeto.

Arquivo Função
participacao_setorial_deploy.qmd Documento Quarto com a interface e o servidor Shiny.
my_workspace_light.RData Base leve carregada pelo aplicativo no deploy.
01_preparar_dados.R Script de replicação e preparação dos dados. Não precisa rodar no deploy, mas deve ficar no repositório para transparência.

O arquivo my_workspace_light.RData deve ser gerado antes da publicação com source("01_preparar_dados.R"). Durante o deploy, o .qmd apenas carrega esse arquivo, reduzindo o tempo de inicialização do aplicativo.
